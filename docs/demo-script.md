# Novel-to-Script 演示视频脚本

> 预计时长：3-4 分钟
> 用途：剪映 AI 配音旁白文案，配合录屏画面使用

---

## 旁白脚本

### 【场景 1：开场 + 登录】（约 20 秒）

**画面**：浏览器打开 localhost:3000 → 登录页面

**旁白**：

大家好，今天演示的是 Novel-to-Script，一个 AI 驱动的小说转剧本工具。
只需要上传一段小说文本，系统就能自动分析出角色、场景和对话节拍，生成结构化的剧本。

我们先登录账号。

---

### 【场景 2：上传页面】（约 30 秒）

**画面**：登录后进入上传首页 → 展示三个特性卡片 → 点击上传区域选择文件

**旁白**：

登录后进入上传页面。页面展示了三个核心能力：多格式解析、智能结构分析、以及结构化输出。

系统支持三种文本格式：TXT、Markdown 和 Word 文档。

我们点击上传区域，选择一篇短篇小说作为示例。

---

### 【场景 3：选择设置 + 开始转换】（约 15 秒）

**画面**：文件上传成功提示 → 展示设置面板 → 选择"电视剧"和"中文" → 点击"开始转换"

**旁白**：

文件已选中。在下方可以选择剧本类型——电视剧、电影、短剧或舞台剧，以及输出语言。

我们选择"电视剧"，然后点击开始转换。

---

### 【场景 4：转换进度】（约 40 秒）

**画面**：进入进度页面 → 左侧 6 阶段进度条逐步推进 → 右侧实时事件日志滚动

**旁白**：

转换正式开始。系统会经历六个阶段：

第一步，解析文件，提取纯文本。
第二步，按章节标记拆分原文。
第三步，调用大模型分析整篇小说，提取角色列表、地点信息、以及故事摘要。
第四步，将每一章按场景分割，并自动对齐场景边界。
第五步，也是最核心的一步——使用 ReAct 智能体逐场景提取对白节拍。
最后一步，组装成完整的 YAML 剧本。

右侧的事件日志展示了每个后端服务的实时调用记录。整个转换大约需要两到五分钟。

---

### 【场景 5：编辑器 + 预览】（约 50 秒）

**画面**：转换完成 → 自动跳转编辑器页面 → 左侧 Monaco YAML 编辑器，右侧剧本预览 → 滚动展示 YAML 内容 → 鼠标在预览面板滚动

**旁白**：

转换完成，自动进入编辑器页面。

左侧是 Monaco 代码编辑器，可以直接编辑生成的 YAML。顶部显示行数和字符数统计。

右侧是实时剧本预览，YAML 中的角色、场景、节拍会自动渲染成可读的剧本格式。

我们来看一下生成的内容。角色部分包含姓名、别名、角色定位和描述。别名关系是系统自动补全的——比如"前女友"和"前男友"是双向关联的。

往下看场景部分，每个场景包含地点、时间、场景描述，以及具体的节拍列表。节拍类型分为五种：动作、对话、场景转换、内心独白和蒙太奇。

对话节拍会标注说话角色，这些归属是系统通过多层推断策略自动确定的。

---

### 【场景 6：导出功能】（约 15 秒）

**画面**：点击右上角导出按钮 → 展示导出菜单 → 选择 YAML 格式下载

**旁白**：

编辑完成后，可以导出为 YAML 格式，方便后续在剧本软件中使用。也支持导出为 Fountain 格式，一种专业的剧本编写格式。

---

### 【场景 7：历史记录】（约 20 秒）

**画面**：点击顶部导航"历史" → 进入历史记录页面 → 展示卡片视图 → 搜索筛选演示 → 点击某条记录的"查看"

**旁白**：

系统会自动保存每次转换记录。在历史页面，可以按卡片或列表两种模式浏览，支持按标题搜索和按剧本类型筛选。

点击任意一条记录，可以直接回到编辑器查看或重新编辑生成的剧本。

---

### 【场景 8：结尾】（约 10 秒）

**画面**：回到上传首页

**旁白**：

以上就是 Novel-to-Script 的完整功能演示。项目基于 Python FastAPI 微服务架构，使用 LangGraph ReAct 智能体进行节拍提取，Pydantic 结构化输出保证格式一致性。

感谢观看。

## 录制建议

| 画面 | 录制方式 | 时长 |
|------|----------|------|
| 登录 | 直接录屏，输入测试账号 | 10s |
| 上传首页 | 录屏，鼠标缓慢移动展示卡片 | 8s |
| 文件上传 | 录屏，拖拽 test-chapters.txt 到上传区 | 5s |
| 设置面板 | 录屏，切换剧本类型 | 5s |
| 开始转换 | 点击按钮，然后暂停录制，等转换实际跑起来再继续 | — |
| 进度页面 | 转换跑起来后录屏 30-60s（加速剪辑到 10s） | 10s |
| 编辑器 | 录屏，鼠标缓慢滚动 YAML 和预览面板 | 30s |
| 导出 | 点击导出按钮 | 8s |
| 历史记录 | 导航到历史页面，展示卡片和搜索 | 15s |
| 结尾 | 回到首页 | 5s |

> **提示**：转换过程中进度页可能需要等待 2-3 分钟。建议：先用手机掐秒表走一遍完整流程确保数据正常，然后正式录制时转换部分录开头和结尾各 10 秒，中间用加速或跳剪处理。

## 剪映操作步骤

1. 打开剪映 PC 版，导入录屏视频
2. 将旁白文案逐段粘贴到「文本」→「新建文本」
3. 每段文本点击「朗读」→ 选择 AI 声音（推荐「解说小明」或「纪录片男声」）
4. AI 生成音频后，拖动音频对齐到对应画面时间轴
5. 转换进度部分加 2-4 倍速
6. 加一段轻量背景音乐（BGM 音量调到 10-15%）
7. 导出 1080p
